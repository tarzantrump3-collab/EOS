To complete this lab, you’ll be working within a Unix-like terminal (Linux or macOS). Follow these steps sequentially to understand how directory navigation, file manipulation, and linking work.

Phase 1: Directory Structure & Navigation
We will build the nested structure and verify our location at each step.

Create the parent directory:
mkdir lab1

Move into it and create the next level:
cd lab1
mkdir dirA

Navigate into dirA and create dirB:
cd dirA
mkdir dirB

Verify your path:
pwd
(This should show something ending in /lab1/dirA)

Check for files (including hidden):
ls -a
(You will see . and .., which represent the current and parent directories.)

Phase 2: Basic File Creation
We’ll scatter three empty files across your new folders.

In dirA: touch empty1.txt

In dirB:
cd dirB
touch empty2.txt

Back in lab1:
cd ../..

Phase 3: File Manipulation (Copy, Move, Delete)
Now, let's work with actual content and organize your files.

Create 5 files with content:
echo "Hello 1" > f1.txt
echo "Hello 2" > f2.txt
echo "Hello 3" > f3.txt
echo "Hello 4" > f4.txt
echo "Hello 5" > f5.txt

Copy them to dirA:
cp f1.txt f2.txt f3.txt f4.txt f5.txt dirA/

Rename files (inside dirA):
cd dirA
mv f1.txt renamed_1.txt
mv f2.txt renamed_2.txt

Delete a file:
rm f5.txt

Phase 4: Hard vs. Symbolic Links
This part demonstrates how the file system tracks data.

Create the source file:
echo "Original Content" > file1.txt

Create a Hard Link:
ln file1.txt hardlink.txt
(A hard link is a second pointer to the same data on the disk.)

Create a Symbolic (Soft) Link:
ln -s file1.txt symlink.txt
(A symlink is a shortcut pointing to the filename file1.txt.)

The Deletion Test:
rm file1.txt

Observe the results:

cat hardlink.txt → Works. The data still exists because the hard link keeps it alive.

cat symlink.txt → Fails. You’ll get a "No such file or directory" error because the shortcut points to a name that no longer exists (a "broken" link).

Cleanup:
unlink hardlink.txt
unlink symlink.txt

It looks like you are exploring the essential Linux commands for viewing and creating file content. While they might seem similar, each serves a specific purpose depending on whether you're creating a quick note or inspecting a massive log file.

1. Creating & Writing Files
touch [filename]: Used primarily to create an empty file or update the timestamp of an existing one.

echo "text": Prints text to the terminal. When combined with redirection (>), it writes that text into a file.

echo "Hello" > file.txt (Overwrites file)

echo "Hello" >> file.txt (Appends to the end of the file)

2. Viewing Content (The "Quick" Way)
cat (Concatenate): Dumps the entire contents of a file into your terminal window.

Best for: Small files.

Downside: If the file has 10,000 lines, it will scroll past your eyes instantly.

head: Shows the first 10 lines of a file.

tail: Shows the last 10 lines of a file.

Pro Tip: Use tail -f to watch a file update in real-time (perfect for log files).

3. Interactive Viewing (Pagers)
When files are too large for cat, we use "pagers" to scroll through them manually.

more: An older utility that allows you to scroll down through a file screen-by-screen. It is primitive and doesn't allow easy backward scrolling.

less: The modern standard. Despite the name, "less is more" because it has more features.

Allows scrolling up and down.

Allows searching (type / followed by your keyword).

Doesn't load the entire file into memory, making it lightning-fast for huge files.

Summary Comparison
Command	Best Use Case	Navigation
cat	Viewing tiny files or merging files.	None (Static)
head	Checking headers or file formats.	Top only
tail	Checking recent errors in logs.	Bottom only
more	Basic page-by-page viewing.	Forward only
less	Reading long documents/logs.	Full (Up/Down/Search)

This set of exercises covers the core toolkit used by system administrators and power users. Here is your step-by-step guide to mastering these utilities.

1. Word Analysis (sort, uniq, wc)
These commands are often "piped" together to clean and count data.

Create the file:
echo -e "apple\nbanana\napple\norange\nbanana\nbanana" > repeats.txt

Count unique words:
sort repeats.txt | uniq -c
(Note: uniq only detects duplicates on adjacent lines, so you must sort first.)

Display only unique entries:
sort repeats.txt | uniq

Count stats (Lines, Words, Chars):
wc repeats.txt

2. Text Processing (grep, cut, tr)
Create data:
echo -e "ID:101:Alice:Admin\nID:102:Bob:User\nID:103:Charlie:User" > users.txt

Search with grep:
grep "User" users.txt

Extract columns with cut:
cut -d':' -f3 users.txt (Extracts the 3rd column, using ":" as the delimiter)

Convert to uppercase with tr:
cat users.txt | tr 'a-z' 'A-Z'

3. Comparing & Patching (diff, patch)
Create two versions:
echo -e "Line 1\nLine 2" > v1.txt
echo -e "Line 1\nLine 2 Modified" > v2.txt

Generate a patch:
diff -u v1.txt v2.txt > changes.patch

Apply the patch to v1:
patch v1.txt < changes.patch

Verify: cat v1.txt (It should now match v2.txt)

4. Relational Data (join)
Create files (Must be sorted!):
echo -e "101 Alice\n102 Bob" > names.txt
echo -e "101 $500\n102 $600" > salaries.txt

Combine them:
join names.txt salaries.txt

5. System Monitoring (ps, time, du, df)
View processes: ps aux or ps -ef

Time a command: time sleep 2 (Tells you how long the command took to run)

Check Disk Usage (Directory): du -sh . (Summarizes current folder size in human-readable format)

Check Disk Free (System): df -h

6. Permissions & Ownership (chmod, chown)
Permissions are represented as User-Group-Others.

Modify Permissions:
touch perm.txt
chmod 755 perm.txt (7=Read/Write/Exec for owner, 5=Read/Exec for others)

Check permissions: ls -l perm.txt

Change Owner/Group:
sudo chown username perm.txt
sudo chgrp groupname perm.txt
(Note: You usually need sudo to change ownership to a different user.)

7. Locating Files (find, xargs)
Find by name: find . -name "*.txt"

Find by size: find . -size +10k (Files larger than 10KB)

Command locations:
which ls (Shows binary path)
whereis ls (Shows binary, source, and manual page paths)

Using xargs with find:
find . -name "*.txt" | xargs grep "pattern"
(This takes the list of files found and passes them as arguments to grep.)
